## Why

The two-shard split (`llm_repos`, `ssh_repos`) is a historical artifact. WS4 merges them into a single namespace. `query_master` ignores shards. The `domain_hint` field carries taxonomy. The split only creates routing confusion (e.g. the `containers/podman` shard mismatch in B6). Collapsing removes a category of errors permanently.

## Step 3: Merge shard-level metadata files

Each shard has `index.yaml`, `graph.yaml`, `trust-gates-report.yaml`, `validate.py`, `progress.yaml`, `audit-progress.yaml`. Handle each:

### graph.yaml

The shard-level `graph.yaml` files contain edges that WS4 already compiles into `master_graph.yaml`. Merge them into a single `repos/knowledge/graph.yaml`:

```python
import yaml
from pathlib import Path

llm = yaml.safe_load(Path('llm_repos/knowledge/graph.yaml').read_text())
ssh = yaml.safe_load(Path('ssh_repos/knowledge/graph.yaml').read_text())

merged = {
    'nodes': (llm.get('nodes') or []) + (ssh.get('nodes') or []),
    'edges': (llm.get('edges') or []) + (ssh.get('edges') or []),
}

Path('repos/knowledge/graph.yaml').write_text(
    yaml.safe_dump(merged, sort_keys=False, allow_unicode=False, default_flow_style=False)
)
print(f"Merged graph: {len(merged['nodes'])} nodes, {len(merged['edges'])} edges")
```

### index.yaml

The ssh shard has a structured index; the llm shard has a different format. Since WS4 compiles `master_index.yaml` by scanning `repos/*.yaml` files (not from shard indexes), the shard-level index is not authoritative. Create a minimal placeholder:

```yaml
# repos/knowledge/index.yaml
# Shard-level index (informational only).
# Authoritative index: master_index.yaml (compiled by WS4).
note: "This file is not consumed by the pipeline. See master_index.yaml."
```

### validate.py

The two validate.py scripts are different implementations. Create a single `repos/knowledge/validate.py` that validates the merged structure. It should:

- Check that every entry in `graph.yaml` references a known repo
- Print PASS/FAIL summary

Base it on the `llm_repos` version (the more complete one), but update all paths to use the single directory.
